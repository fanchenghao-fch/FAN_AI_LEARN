## 1. 项目概述与愿景

### 1.1 一句话描述

> **把任何知识，变成一场你停不下来的闯关游戏。**

### 1.2 核心价值主张

| 痛点 | 解决方案 |
|------|---------|
| 学知识枯燥，难以坚持 | 闯关游戏化，即时反馈，让学习像刷短视频一样上瘾 |
| 找学习资料费时费力 | AI 一键生成题目，无需自己整理 |
| 学完就忘，缺少复盘 | 自动生成知识总结 + 智能错题本 + 遗忘曲线提醒 |
| 学习孤独，缺少动力 | 好友PK、排行榜、分享报告、学习圈子 |

### 1.3 产品愿景

成为**知识学习的 "AI 引擎"** —— 无论你想学什么，只需输入一句话 / 一个文档 / 一个链接，AI 就能自动生成一套高质量的交互式学习体验。覆盖 K12 学科、大学课程、职业技能、考公考研、企业培训、兴趣爱好等全部学习场景。

### 1.4 为什么选择微信小程序

| 维度 | 优势 |
|------|------|
| **即用即走** | 无需下载安装，扫码或搜索即可开始学习 |
| **流量入口** | 微信 13亿+ 月活，朋友圈/群聊/公众号天然传播渠道 |
| **社交裂变** | 分享闯关成绩单、PK 邀请，自带病毒传播属性 |
| **生态能力** | 微信支付（VIP 变现）、订阅消息（留存）、云开发（低成本后端） |
| **合规性** | 微信内容审核机制帮助过滤不良内容 |

## 2. 市场分析

### 2.1 市场规模

- **中国在线教育市场**：2025 年规模约 6000 亿元，年均增速 15%+
- **微信小程序教育类目**：日活用户超 2 亿，教育类小程序数量年增 40%+
- **AI+教育赛道**：2025-2026 年最热门赛道之一，资本重点关注

### 2.2 市场机会

当前市场存在一个明显的**空白地带**：

```
现有产品光谱：
  传统题库类 ────────────── 已有 AI 教育类 ────────────── 通用 AI 助手
  （驾考宝典、      （Duolingo Max、        （ChatGPT、
   粉笔公考）         Khan Academy AI）       豆包、Kimi）
      │                      │                       │
      │         ★ 我们的位置 ★                     │
      │     「AI 生成 + 游戏化闯关                 │
      │        + 全知识领域覆盖                    │
      │        + 微信生态内」                      │
      │                                           │
  内容固定，         有 AI 但领域              通用但缺乏学习
  领域单一           垂直/门槛高               场景化设计
```

**核心差异化**：没有人把 "任意知识 → AI 出题 → 游戏化闯关 → 社交分享" 这条链路在微信生态里做透。

### 2.3 目标用户分层

| 用户层 | 占比预估 | 典型场景 | 付费意愿 |
|--------|---------|---------|---------|
| **学生群体**（12-22岁） | 35% | 背单词、期末复习、考研/考公 | 中（有家长付费） |
| **职场人群**（22-40岁） | 40% | 面试刷题、行业知识、考证 | 高 |
| **终身学习者**（40+岁） | 15% | 兴趣爱好、健康养生、历史人文 | 中 |
| **企业/B端** | 10% | 员工培训、合规考试 | 极高 |

## 3. 用户画像

### 3.1 核心用户画像

#### 👩‍🎓 小李 — 大学生
> "期末要考 6 门课，笔记太乱不想整理，想找人一起刷题但找不到。"

- **场景**：期末复习，把课程 PPT/教材章节喂给 AI，自动生成选择题/判断题
- **痛点**：自制题库效率低、一个人刷题无聊、不知道自己掌握程度
- **我们的价值**：AI 30 秒出题 + 闯关排名 + 错题复盘

#### 👨‍💼 老王 — 互联网从业者
> "想转行做产品经理，网上的面试题质量参差不齐，想系统刷题但没时间整理。"

- **场景**：输入 "产品经理面试高频题"，AI 搜索最新面经并生成闯关题库
- **痛点**：面试题零散、没有系统学习路径、缺乏模拟面试体验
- **我们的价值**：联网搜索最新题库 + 模拟面试模式 + 答题分析

#### 👵 张阿姨 — 退休人士
> "想学养生知识，但文章太长看不进去，有没有简单好玩的学知识方式？"

- **场景**：输入 "春季养生知识"，AI 生成 10 道趣味问答
- **痛点**：长篇内容阅读困难、记忆效果差、缺乏互动
- **我们的价值**：语音读题 + 大字模式 + 趣味问答 + 分享到广场舞群

#### 🏢 某公司 HR — 培训负责人
> "新员工入职培训每次都要人工出考题，又要批改，太累了。"

- **场景**：上传公司规章制度文档，AI 自动生成合规考试题库
- **痛点**：出题费时、人工批改、无法追踪培训效果
- **我们的价值**：知识库管理 + 自动出题 + 考试模式 + 数据看板

## 4. 核心功能需求（MVP）

### 4.1 功能全景图

```
┌─────────────────────────────────────────────────────┐
│                    MVP 功能模块                        │
├─────────────┬─────────────┬─────────────┬─────────────┤
│  知识输入    │  AI 出题     │  闯关答题    │  学习复盘    │
├─────────────┼─────────────┼─────────────┼─────────────┤
│ · 文本输入   │ · 智能搜索   │ · 选择题     │ · 成绩单     │
│ · 文档上传   │ · 题目生成   │ · 判断题     │ · 错题本     │
│ · 链接解析   │ · 答案验证   │ · 即时反馈   │ · 知识总结   │
│ · 语音输入   │ · 知识溯源   │ · 知识讲解   │ · 分享海报   │
├─────────────┴─────────────┴─────────────┴─────────────┤
│                    用户系统 + 历史记录                   │
└─────────────────────────────────────────────────────┘
```

### 4.2 模块详细说明

#### 4.2.1 知识输入模块

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| **文字输入** | 用户输入一句话/一段话描述想学的知识。例如："我想学习中国近代史的重要事件" | P0 |
| **文档上传** | 支持 PDF、Word、PPT、TXT，AI 解析内容后出题。上限 10MB | P0 |
| **链接解析** | 粘贴网页 URL / 公众号文章链接，AI 抓取内容后出题 | P1 |
| **图片识别** | 拍照/截图上传（如课本页、笔记），OCR 识别后出题 | P1 |
| **视频解析** | 上传视频/粘贴视频链接，AI 提取字幕/语音转文字后出题 | P2 |
| **语音输入** | 直接说出想学的知识，语音转文字后出题（尤其适合老年用户） | P2 |

#### 4.2.2 AI 出题模块

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| **联网搜索** | 当用户输入的知识点不够具体时，AI 自动搜索最新相关内容扩充知识库 | P0 |
| **题目生成** | 基于知识内容自动生成 N 道选择题/判断题，每道题 4 个选项 | P0 |
| **难度分级** | 自动评估题目难度（简单/中等/困难），或允许用户指定难度 | P0 |
| **题目数量** | 默认 10 题，用户可自定义 5-50 题 | P0 |
| **题型混合** | 选择题为主 + 判断题 + 填空题（P2） | P0 |
| **答案校验** | AI 自我验证正确答案的准确性，标注知识来源 | P1 |
| **图片生成** | 为适合配图的题目（如背单词、识图）自动生成配图 | P2 |
| **题目预览** | 出题后允许用户预览和手动调整（删除/修改某题）再开始闯关 | P1 |

#### 4.2.3 闯关答题模块

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| **闯关流程** | 逐题展示，选择答案后立即显示正误 | P0 |
| **即时反馈** | ✅ 答对：绿色动画 + 鼓励语 + 知识要点回顾 | P0 |
| **即时讲解** | ❌ 答错：红色标记 + 正确答案高亮 + 详细知识讲解 | P0 |
| **进度条** | 实时显示当前进度（如 3/10）和正确率 | P0 |
| **计时器** | 可选：每题限时作答（增加紧迫感），也可不限时 | P1 |
| **跳题/回看** | 允许跳过不确定的题，最后统一回看；也支持回到上一题 | P1 |
| **音效反馈** | 答对/答错不同音效，增强游戏感 | P1 |
| **连击系统** | 连续答对有 combo 特效（如 🔥连击×5），激励用户 | P1 |
| **生命值** | 可选模式：总共 N 条命，答错扣命，0 命通关失败（增加挑战性） | P2 |
| **AI 追问** | 答错后 AI 生成一道类似题追问，确保真正掌握 | P2 |

#### 4.2.4 学习复盘模块

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| **通关结果** | 展示得分、正确率、用时、排名（如有排行榜） | P0 |
| **知识总结** | AI 生成该知识领域的核心要点总结（3-5 条精华） | P0 |
| **错题回顾** | 列出所有答错的题 + 正确答案 + 详细解析 | P0 |
| **掌握度雷达图** | 可视化展示知识子领域的掌握情况 | P1 |
| **分享海报** | 自动生成精美海报（含分数、知识领域、一句金句），可分享到朋友圈/群 | P0 |
| **学习建议** | AI 根据答题表现，推荐下一步学习方向 | P1 |

### 4.3 用户系统（MVP）

| 功能点 | 描述 |
|--------|------|
| **微信一键登录** | 通过 `wx.login` 获取用户身份，无需额外注册 |
| **学习历史** | 记录所有闯关记录：时间、知识领域、题目数、得分 |
| **我的错题本** | 聚合所有错题，支持按知识领域筛选和重新作答 |
| **收藏/书签** | 收藏感兴趣的闯关游戏，方便回看 |

## 6. 亮点功能创意

> 以下是我结合行业趋势和用户心理提出的**差异化亮点功能**，可以作为产品出圈的核心竞争力。

### 🌟 亮点 1：「AI 学习伴侣」— 拟人化学习伙伴

**概念**：不是冷冰冰的答题页面，而是一个有性格的 AI 学习伙伴陪你闯关。

- 用户可选择不同的 AI 伙伴形象（学霸猫、智慧老人、书虫机器人等）
- AI 伙伴有情绪反馈：你答对时它会兴奋，答错时它会鼓励你，连击时它会惊叹
- AI 伙伴会记住你的学习风格，比如"你上次在这个知识点上卡住了，我再帮你巩固一下"
- **走红潜力**：萌宠/角色 IP 自带传播属性，截图分享欲强

### 🌟 亮点 2：「知识对战」— 微信好友实时 PK

**概念**：直接利用微信社交关系链，发起知识挑战。

- 发起人选定知识领域 → 生成对战题目库 → 微信分享邀请卡片 → 好友点击进入对战
- 同屏实时 PK（类似头脑王者），看谁正确率高
- 对战结束生成双方对比报告，可分享到朋友圈"叫板"
- **走红潜力**：社交裂变的天然引擎，类似当年的"头脑王者"但知识覆盖面更广

### 🌟 亮点 3：「一分钟学会 X」— AI 短视频化学习

**概念**：用短视频讲述 + 闯关验证的模式，适配碎片化学习。

- AI 将知识点拆解为 1 分钟的短视频脚本 + 自动配音
- 看完视频 → 立刻 3 题闯关验证 → 分享「一分钟学会XXX」成就卡
- 覆盖场景：通勤、等餐、上厕所——"随手学一点"
- **走红潜力**：短视频是当前最高频的内容消费形式，降低学习门槛

### 🌟 亮点 4：「知识星球」— 可视化知识图谱

**概念**：用户学习的每一个知识点都变成一颗"星球"，关联知识点形成星系。

- 每个用户拥有自己的"知识星球"，学得越多、星球越亮
- 知识点之间有引力连线（如 "Python" 关联到 "机器学习"）
- AI 推荐"你的知识星系还缺一颗 X 星，要探索吗？"
- **走红潜力**：高度可视化 + 收藏/养成心理，用户愿意截图分享

### 🌟 亮点 5：「AI 模拟面试」— 垂直场景深度方案

**概念**：针对高价值场景（求职/考研复试），提供沉浸式 AI 模拟面试。

- AI 扮演面试官，基于岗位 JD 和常见面试题进行提问
- 用户语音/文字回答 → AI 评估答案质量 → 给出改进建议
- 模拟面试结束后生成评分报告（沟通能力、专业度、逻辑性等维度）
- **变现潜力**：高频刚需 + 高付费意愿（参考：模拟面试单次 100-300 元市场价）

### 🌟 亮点 6：「拖延症克星」— 游戏化学习习惯养成

**概念**：用游戏的机制对付学习的拖延。

- **每日能量条**：每天有固定的"学习能量值"，用完即止（饥饿营销）
- **学习连胜**：连续学习天数形成 streak，有皇家匹配特效，断签有"续命卡"
- **Boss 战**：每完成一个知识模块，解锁一个 Boss 战（综合大考）
- **宝箱掉落**：随机掉落"双倍经验卡""免答卡""换题卡"等道具
- **走红潜力**：游戏化是提升留存的最有效手段，参考 Duolingo 的成功

### 🌟 亮点 7：「知识胶囊」— AI 时间胶囊

**概念**：用户设定一个学习目标，AI 定期推送学习内容。

- 比如"30 天学会 Python 基础" → AI 每天推送一个知识点 + 5 道题
- 每 7 天一次阶段性测验，30 天后大考
- 学完开启"时间胶囊"——看到第一天 vs 第三十天的能力变化
- **走红潜力**：仪式感强，用户愿意分享"毕业证书"到朋友圈

### 🌟 亮点 8：「万物可题」— 多模态输入零门槛

**概念**：真正做到"看到什么，就能学什么"。

- **拍课本** → OCR+AI → 自动出题
- **拍路牌/菜单/说明书** → AI 识别 → 趣味问答
- **录音一段话**（如会议内容）→ AI 提取要点 → 出题验证理解
- **拍一道你不会的题** → AI 不仅解答，还生成 5 道同类题帮你练习
- **走红潜力**：极低的使用门槛，"拍一拍就能学"本身就有传播价值

### 8.2 关键技术决策建议

| 决策点 | 建议方案 | 理由 |
|--------|---------|------|
| **后端框架** | Node.js (NestJS) 或 Python (FastAPI) | Node.js 生态适合高并发 I/O；Python AI 生态最成熟 |
| **数据库** | MySQL（用户/订单）+ MongoDB（题库/文档） | 关系型管账号交易，文档型存非结构化题库内容 |
| **缓存** | Redis | 缓存热门题库、用户会话、排行榜 |
| **文件存储** | 腾讯云 COS / 阿里云 OSS | 存储用户上传的文档和 AI 生成的图片 |
| **消息队列** | RabbitMQ / Kafka | AI 出题异步处理，削峰填谷 |
| **微信云开发** | 可考虑作为 MVP 阶段快速验证 | 降低运维成本，但长期扩展性受限 |
| **WebSocket** | 用于实时 PK 对战 | Socket.io 或微信原生 WebSocket |

### 8.3 内容安全架构（关键）

微信小程序对内容审核非常严格，AI 生成的内容必须过安全检测：

```
用户输入 → 微信自带内容安全 API（msgSecCheck）
         → 第三方敏感词过滤
         → AI 生成内容 → 再次安全检测
         → 人工审核队列（高风险内容）
```

## 11. 风险分析与应对策略

### 11.1 风险矩阵

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| **AI 生成内容不准确（幻觉）** | 🔴 高 | 🟡 中 | 题目校验层 + 知识来源标注 + 用户反馈机制 |
| **微信内容合规审核** | 🔴 高 | 🟡 中 | 多层内容过滤 + 敏感词库 + 人工审核队列 |
| **AI API 服务中断** | 🟡 中 | 🟡 中 | 多模型备份 + 题目缓存 + 降级方案（题库模式）|
| **用户增长不及预期** | 🟡 中 | 🟡 中 | 社交裂变设计 + 微信广告投放 + KOL 合作 |
| **竞品快速跟进** | 🟡 中 | 🔴 高 | 快速迭代 + 社区壁垒 + B端差异化 |
| **版权风险（题目/内容）** | 🟡 中 | 🟢 低 | 不直接存储第三方原文 + 仅保留 AI 生成的题目 |
| **数据隐私** | 🔴 高 | 🟢 低 | 用户上传文档及时清理 + 数据加密 + 隐私协议 |
| **用户付费意愿低** | 🟡 中 | 🟡 中 | 免费额度体验够好 + 痛点场景深度覆盖 + 阶梯定价 |

### 11.2 关键风险深度分析

#### 风险 1：AI 幻觉导致错误知识传播

这是**最核心的风险**。如果 AI 生成的答案有误，用户学到了错误知识，对教育产品是致命的。

**多层防线**：
1. **第一层**：选择推理能力最强的大模型（Claude Opus / GPT-4o 级别）
2. **第二层**：用另一个模型（或同模型不同 prompt）校验每道题的正确答案
3. **第三层**：每道题标注"知识来源"，让用户知道信息出处
4. **第四层**：用户举报入口（"这道题有错误"），积累反馈数据优化 prompt
5. **第五层**：对于高价值题库（考公、考证），引入人工审核

#### 风险 2：微信内容审核

微信小程序的教育类目对 AI 生成内容的审核趋严。尤其在政治、历史、医疗等领域。

**应对**：
- 接入微信 `security.msgSecCheck` 和 `security.mediaCheckAsync`
- 建立领域敏感词库，特定领域（如现代史、医疗）的题目走更严格审核
- 用户输入和 AI 输出都过安全检测
- 准备好应对微信审核的运营团队

## 13. 成功指标与度量

### 13.1 北极星指标

> **每周完成的闯关数**（core loop completion）

这直接反映了用户是否真正在使用产品学习。

### 13.2 核心指标

| 指标 | 意义 | MVP 阶段目标 | 增长阶段目标 |
|------|------|-------------|-------------|
| **DAU** | 日活跃用户 | 100+ | 5,000+ |
| **日均闯关完成数** | 核心行为 | 150+ | 10,000+ |
| **次日留存率** | 用户粘性 | 30%+ | 40%+ |
| **7 日留存率** | 长期价值 | 15%+ | 25%+ |
| **分享率** | 病毒传播 | 15%+ | 20%+ |
| **AI 出题成功率** | 技术质量 | 95%+ | 98%+ |
| **题目准确率（用户反馈）** | 内容质量 | 90%+ | 95%+ |
| **用户平均答题时长** | 沉浸度 | 5 分钟+ | 8 分钟+ |

### 13.3 数据埋点计划

```
关键事件埋点：
├── 启动 → 小程序启动次数
├── 输入知识 → 输入方式（文字/文档/链接/语音）
├── 出题开始 → 知识领域、出题数量、等待时长
├── 出题成功/失败 → 成功率、失败原因
├── 答题开始 → 知识领域、题目数量
├── 单题作答 → 题号、是否正确、作答时间
├── 闯关完成 → 总分、正确率、用时、是否分享
├── 分享行为 → 分享渠道（朋友圈/群/好友）、点击率
├── 回访 → 次日是否回访、7日是否回访
└── 付费 → 付费页面曝光、点击、转化
```

## 14. 附录：竞品简要分析

### 14.1 直接竞品

| 产品 | 定位 | 优势 | 劣势 | 我们的差异化 |
|------|------|------|------|-------------|
| **Quizlet** | 背单词/闪卡 | 用户基数大、成熟 | 需手动创建卡片、无 AI 生成 | AI 自动出题，零门槛 |
| **Kahoot!** | 课堂互动答题 | 教育场景深耕 | 侧重课堂、非个人学习 | 自由学习，任何知识 |
| **头脑王者** | 微信答题对战 | 微信生态内、社交强 | 题库固定、非学习导向 | AI 生成 + 真学习 |
| **Duolingo** | 语言学习 | 游戏化极致 | 仅限语言、无开放知识 | 全知识领域 + 中文优先 |

### 14.2 间接竞品

| 产品 | 交集 | 威胁程度 |
|------|------|---------|
| **豆包 / Kimi / 元宝** | 通用 AI 助手，用户可能用它们生成题目 | 🟡 中 |
| **百度文库 AI** | 文档 + AI 出题 | 🟡 中 |
| **知乎 / 得到** | 知识社区，可能拓展 AI 问答功能 | 🟢 低 |
| **驾考宝典 / 粉笔** | 垂直题库，如果开放 AI 出题 | 🟡 中 |

### 14.3 我们的竞争壁垒

1. **数据飞轮**：用户答题数据持续优化 AI 出题质量，题目越用越准
2. **题库沉淀**：AI 生成 + 人工校验的高质量题库形成内容壁垒
3. **社交关系链**：微信生态内的 PK 关系和分享网络
4. **B端客户粘性**：企业接入知识库后迁移成本高
5. **品牌认知**：率先占据"AI学习闯关"的心智定位

## 附录 A：关键术语表

| 术语 | 说明 |
|------|------|
| **闯关** | 一次完整的答题过程，包含 N 道题目，用户逐题作答 |
| **知识领域** | 用户学习的知识范畴，如"Python编程""中国近代史" |
| **知识库** | B端功能，企业/个人上传的文档集合，作为出题素材 |
| **题目校验** | AI 二次验证题目和答案的准确性 |
| **连击（Combo）** | 用户连续答对题目数量的计数，用于增强游戏感 |
| **生命值** | 游戏化模式下用户的容错次数 |
