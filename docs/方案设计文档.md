## 1. 技术选型

### 1.1 前端技术

> **注**：Phase 0 阶段使用 React SPA（Web 原型）快速验证核心流程，前端 UI 细节后续会单独设计网页原型。Phase 1 起迁移到 Taro 微信小程序。

#### Phase 0：Web 原型

| 技术 | 选型 | 理由 |
|------|------|------|
| 框架 | **React 18 + TypeScript** | 与后续 Taro 小程序共享 Hooks/状态管理/类型定义 |
| 构建工具 | **Vite** | 极速 HMR，开发体验最佳 |
| 路由 | **React Router v6** | SPA 页面导航 |
| UI 样式 | 纯 CSS / Tailwind CSS | 原型阶段快速搭建，不强依赖组件库 |

#### Phase 1+：微信小程序

| 技术 | 选型 | 理由 |
|------|------|------|
| 框架 | **Taro 4.x + React 18** | React 生态复用、TypeScript 支持业界最佳、标准 npm 工程化 |
| 状态管理 | **Zustand** | ~1KB 极轻量、Taro 完美兼容 |
| UI 组件库 | **NutUI（Taro 版）** | 京东官方出品，与 Taro 同团队 |
| 海报生成 | **Painter** | JSON 配置驱动，适合动态海报 |

> **选 Taro 4.x 不选 uni-app/原生的简要理由**：
> - React 生态复用（Web 原型的 Hooks/Zustand/Types 可直接迁移到小程序）
> - TypeScript 类型定义完善（本项目数据结构复杂）
> - VSCode 标准工作流（非 HBuilderX 强绑定）
> - **Taro 4.x 新特性**：`CompileMode` 混合编译模式——简单组件编译时处理、复杂组件运行时处理，大幅缩小与 uni-app 的性能差距；原生支持 Vite 编译，与 Phase 0 Web 原型共享构建工具链；新增鸿蒙端平台支持，为未来扩展留空间

### 1.2 后端框架

**推荐：Python FastAPI**

| 维度 | FastAPI | Node.js (NestJS) | Go (Gin) |
|------|---------|-------------------|----------|
| **AI/LangChain 生态** | ⭐⭐⭐⭐⭐ 原生 | ⭐⭐⭐ JS 版 LangChain | ❌ 无 |
| **异步支持** | ⭐⭐⭐⭐⭐ asyncio | ⭐⭐⭐⭐⭐ 原生 | ⭐⭐⭐⭐⭐ |
| **类型安全** | ⭐⭐⭐⭐ Pydantic v2 | ⭐⭐⭐⭐ TypeScript | ⭐⭐⭐ |
| **性能** | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **学习曲线** | 低 | 中 | 中高 |
| **社区生态** | 极活跃 | 极活跃 | 活跃 |

**选 FastAPI 的核心理由**：
- Python 是 AI/ML 生态的唯一一等公民，LangChain、OpenAI SDK、HuggingFace 等全是 Python 原生
- Pydantic v2 提供与 LangChain `StructuredOutputParser` 完美兼容的类型校验
- `asyncio` 天然适合 AI API 调用场景（大部分时间是等待网络 IO）
- FastAPI 自带 Swagger 文档，Phase 0 前后端联调效率极高

**数据库 & 缓存**：

| 组件 | 选型 | 说明 |
|------|------|------|
| 关系数据库 | **PostgreSQL** | 用户、订单、知识库等结构化数据 |
| 向量数据库 | **pgvector**（PostgreSQL 扩展） | RAG 语义检索，无需额外部署 |
| 缓存 | **Redis** | 热门题目缓存、会话管理、排行榜 |
| 文件存储 | **MinIO**（自建）/ 阿里云 OSS | 文档上传、海报图片 |
| 任务队列 | **Celery + Redis** | AI 出题异步处理（长文档场景） |

### 1.3 AI 编排框架：LangChain

这是本次方案设计的**核心差异点**。LangChain 作为 AI 编排层，统一管理所有与大模型的交互逻辑。

#### LangChain 在本项目中的角色

```
┌─────────────────────────────────────────────────────────┐
│                    LangChain 编排层                       │
│                                                         │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐      │
│  │ Prompt 管理  │  │ 结构化输出   │  │  模型路由    │      │
│  │ (ChatPrompt │  │ (StructOut  │  │ (model      │      │
│  │  Template)  │  │  putParser) │  │  router)    │      │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘      │
│         │                │                │              │
│  ┌──────┴────────────────┴────────────────┴──────┐      │
│  │              Chain / RunnableSequence          │      │
│  │         (出题链 / 校验链 / 分析链)               │      │
│  └───────────────────────┬───────────────────────┘      │
│                          │                               │
│  ┌───────────────────────┴───────────────────────┐      │
│  │              RAG 管线（Phase 2）                │      │
│  │  document_loaders → text_splitter → vector     │      │
│  │  _store → retrieval_chain → 增强出题           │      │
│  └───────────────────────────────────────────────┘      │
│                          │                               │
│  ┌───────────────────────┴───────────────────────┐      │
│  │              可观测性（LangFuse）               │      │
│  │         token 追踪 / 延迟监控 / 质量评估        │      │
│  └───────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────┘
```

#### LangChain 核心组件使用规划

| LangChain 组件 | 本项目用途 | Phase |
|------|------|:---:|
| `ChatPromptTemplate` | 统一管理出题/校验/分析三套 Prompt 模板 | P0 |
| `StructuredOutputParser` + Pydantic | 确保 AI 输出严格符合 `Question[]` / `QuizResult` 结构 | P0 |
| `ChatOpenAI` (兼容模式) | 封装 DeepSeek V4 API 调用（兼容 OpenAI 协议） | P0 |
| `RunnableSequence` (LCEL) | 构建 "模板填充→模型调用→结构化解析" 处理链 | P0 |
| `model_router` | 按任务类型分发到不同模型（出题→Pro，校验→Flash） | P0 |
| `document_loaders` (PyPDF/Unstructured) | 解析上传的 PDF/Word/PPT 文档 | P2 |
| `text_splitter` (RecursiveCharacter) | 文档分块，为 RAG 做准备 | P2 |
| `vector_store` (pgvector) | 文档向量化存储 + 语义检索 | P2 |
| `RetrievalQA` / `create_retrieval_chain` | 基于知识库的增强出题 | P2 |
| LangFuse callback | Token 消耗追踪、延迟监控、质量评估 | P1 |

#### LangChain 中的模型路由策略

```python
# 伪代码示例：LangChain 模型路由
from langchain_deepseek import ChatDeepSeek

pro_model = ChatDeepSeek(model="deepseek-v4-pro", temperature=0.7)
flash_model = ChatDeepSeek(model="deepseek-v4-flash", temperature=0.3)

# 按任务选择模型
model_router = {
    "quiz_generation": pro_model,     # 出题需要强推理
    "quiz_validation": flash_model,   # 校验需要快速低成本
    "result_analysis": pro_model,     # 分析报告需要高质量

}
```

## 3. LangChain 实现方案（重点）

### 3.1 出题链（QuizGenerationChain）

这是产品的核心——将用户输入转化为结构化题目列表。

```python
from langchain_deepseek import ChatDeepSeek
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser, JsonOutputParser
from langchain_core.pydantic_v1 import BaseModel, Field
from typing import List, Optional, Literal

# ── 1. 定义输出结构（与前端共享类型定义）──
class Option(BaseModel):
    key: Literal["A", "B", "C", "D"]
    text: str

class Question(BaseModel):
    id: str = Field(description="题目唯一标识，如 q1, q2")
    type: Literal["choice", "truefalse", "fill"]
    content: str = Field(description="题目内容")
    options: List[Option] = Field(description="选项列表")
    correct_answer: str = Field(description="正确答案")
    explanation: str = Field(description="详细解析，80-150字")
    source: Optional[str] = Field(description="知识来源")
    difficulty: Literal["easy", "medium", "hard"]

class QuizOutput(BaseModel):
    title: str = Field(description="闯关标题，简洁有趣，15字以内")
    knowledge_domain: str = Field(description="知识领域分类")
    questions: List[Question] = Field(description="题目列表")

# ── 2. 定义 Prompt 模板 ──
QUIZ_GENERATION_PROMPT = ChatPromptTemplate.from_messages([
    ("system", """## 角色
你是一位资深的考试命题专家和教育测量学研究者。你擅长根据任意知识内容，
设计具有区分度、无歧义、教学价值高的选择题。

## 核心要求
1. 每题 4 个选项（A/B/C/D），有且仅有一个正确答案
2. 难度分布：简单 30%、中等 50%、困难 20%
3. 干扰项应具有迷惑性（常见误解、相近概念、易混淆的知识点）
4. 题目应覆盖知识内容的主要要点
5. 每道题附带详细解析（80-150字）
6. 标注知识来源

## 输出格式
严格按照 JSON 格式输出，不要输出其他内容。
{format_instructions}
"""),
    ("user", """## 知识内容
{knowledge_input}

## 出题要求
- 题目数量：{question_count} 道
- 题型：{question_types}
- 用户指定难度：{difficulty}
""")
])

# ── 3. 构建 Chain ──
def create_quiz_generation_chain(model_name: str = "deepseek-v4-pro"):
    # 模型
    llm = ChatDeepSeek(
        model=model_name,
        temperature=0.7,          # 适度创造性，保证题目多样性
        max_tokens=4096,
        streaming=True             # 开启流式
    )

    # 结构化输出解析器
    parser = JsonOutputParser(pydantic_object=QuizOutput)

    # LCEL 链式组合
    chain = (
        {
            "knowledge_input": lambda x: x["knowledge_input"],
            "question_count": lambda x: x.get("question_count", 10),
            "question_types": lambda x: ", ".join(x.get("question_types", ["选择题"])),
            "difficulty": lambda x: x.get("difficulty", "auto"),
            "format_instructions": lambda _: parser.get_format_instructions(),
        }
        | QUIZ_GENERATION_PROMPT
        | llm
        | parser                          # 自动解析 JSON → QuizOutput
    )

    return chain
```

### 3.4 模型路由

按任务类型和成本预算自动选择模型：

```python
from enum import Enum

class TaskType(Enum):
    QUIZ_GENERATION = "quiz_generation"      # 题目生成 → Pro
    QUIZ_VALIDATION = "quiz_validation"      # 题目校验 → Flash
    RESULT_ANALYSIS = "result_analysis"      # 分析报告 → Pro
    KNOWLEDGE_SEARCH = "knowledge_search"    # 知识搜索 → Qwen-Plus
    DOCUMENT_PARSE = "document_parse"        # 长文档解析 → Kimi

MODEL_MAP = {
    TaskType.QUIZ_GENERATION: "deepseek-v4-pro",
    TaskType.QUIZ_VALIDATION: "deepseek-v4-flash",
    TaskType.RESULT_ANALYSIS: "deepseek-v4-pro",
    TaskType.KNOWLEDGE_SEARCH: "qwen-plus",
    TaskType.DOCUMENT_PARSE: "kimi-k2-5",       # Phase 2
}

def get_model_for_task(task: TaskType) -> ChatDeepSeek:
    model_name = MODEL_MAP[task]
    return ChatDeepSeek(
        model=model_name,
        temperature=0.1 if task == TaskType.QUIZ_VALIDATION else 0.7,
        streaming=True,
    )
```

# main.py
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
from langchain_core.messages import HumanMessage
import json

app = FastAPI(title="AI Quiz Platform API")

@app.post("/api/quiz/generate")
async def generate_quiz(request: QuizGenerateRequest):
    """
    生成题目 - SSE 流式响应
    """
    chain = create_quiz_generation_chain()

    async def event_stream():
        # 阶段1：搜索
        yield f"event: progress\ndata: {json.dumps({'stage': 'searching', 'message': '正在搜索相关知识...'})}\n\n"

        # 阶段2：生成
        async for chunk in chain.astream(request.model_dump()):
            yield f"event: progress\ndata: {json.dumps({'stage': 'generating', 'chunk': chunk})}\n\n"

        # 阶段3：完成
        yield f"event: done\ndata: {json.dumps({'status': 'completed'})}\n\n"

    return StreamingResponse(
        event_stream(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",
        }
    )

## 8. 内容安全合规（简要）

> **注**：此部分非 Phase 0 重点，功能实现优先。小程序提审前再正式接入微信内容安全 API。

- 敏感领域（政治/医疗/法律）的题目生成需人工审核
