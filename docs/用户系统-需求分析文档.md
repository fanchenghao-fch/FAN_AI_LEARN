## 1. 背景与目标

> **技术备注**：本次迭代数据库选用 **MySQL**（异步驱动 `asyncmy`），原因是配置更简单、本地开发更轻量，且项目中关系型数据无需 PostgreSQL 的高级特性（如 pgvector）。原方案设计文档中 Phase 2 的 RAG 向量检索在迁移到 MySQL 后，可改用内存向量（ChromaDB 嵌入式）或单独的向量服务。

### 1.1 背景

MVP v1.0.0 已完成核心答题闭环（知识输入 → AI 出题 → 闯关答题 → 结果复盘），但缺少用户系统。所有用户数据均为内存态，无法持久化。具体缺失能力：

- 无法识别"谁"在答题
- 无法查看历史学习记录
- 无法沉淀错题供复习
- 无积分/等级体系激励用户持续使用
- 微信小程序端无法获取用户身份（openid/unionid）

### 1.2 目标

在 MVP v1.0.0 基础上，建立完整的轻量级用户系统，实现：

1. **用户身份**：支持微信授权登录（小程序端）+ Mock 登录（H5 开发端）
2. **数据持久化**：闯关记录、错题数据入库
3. **个人中心**：展示学习统计、金币/等级、历史记录
4. **错题复习**：错题聚合、按领域分类、支持重做

### 1.3 平台策略

| 平台 | 登录方式 | 说明 |
|------|---------|------|
| 微信小程序 | `wx.login` → 后端换取 openid | 正式用户入口 |
| H5 Web | Mock 登录（输入昵称） | 开发调试 / 演示用 |

## 3. P0：用户认证与基础身份

> **优先级**：🔴 最高 — 是所有后续功能的前置依赖
> **参考原型**：无独立原型图（简洁登录页自行设计）

### 3.1 微信小程序登录

**流程**：

```
用户点击"我的"Tab
  → 检测登录态（本地 Token 是否存在 & 有效）
  → 若未登录：
      → 调用 wx.login() 获取 code
      → 后端用 code 向微信服务器换取 openid / unionid / session_key
      → 若为新用户：自动注册（生成 UUID、默认昵称"灯灯学员"、0金币）
      → 若为老用户：匹配已有账号
      → 返回 JWT Token → 前端存储到本地
  → 若已登录：
      → 直接进入个人中心
```

**功能要点**：
- 首次登录自动注册，无需用户填写任何信息
- 支持获取微信头像和昵称（`wx.getUserProfile` 或 `<button open-type="chooseAvatar">`）
- JWT Token 有效期 7 天，过期自动重新登录

### 3.2 H5 Mock 登录

**流程**：

```
用户点击"我的"Tab
  → 弹出简洁登录框：输入昵称（可选填头像URL）
  → 后端创建/匹配用户（以昵称为标识）
  → 返回 JWT Token → 前端存储到 localStorage
```

**功能要点**：
- 纯开发/演示用途，不设密码
- 输入昵称即登录，若昵称已存在则匹配已有账号
- 同样返回 JWT，保持前后端鉴权逻辑一致

### 3.3 JWT 鉴权

**后端**：
- 所有用户相关 API（`/api/user/*`）校验 `Authorization: Bearer <token>`
- 未登录或 Token 过期返回 `401 Unauthorized`
- 核心出题 API（`/api/quiz/*`）**不强制要求登录**（不登录也能答题），但登录后自动关联用户

**前端**：
- 请求拦截器自动附加 Token
- 401 响应自动清除本地 Token 并引导重新登录

### 3.4 用户基础信息

每个用户至少包含以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 用户唯一标识 |
| `open_id` | string | 微信 openid（小程序登录用户） |
| `union_id` | string? | 微信 unionid（如有） |
| `nickname` | string | 用户昵称 |
| `avatar_url` | string? | 头像 URL |
| `coins` | int | 金币数量，默认 0 |
| `level` | int | 用户等级，默认 1 |
| `experience` | int | 经验值，默认 0 |
| `created_at` | datetime | 注册时间 |
| `updated_at` | datetime | 最后更新时间 |

## 4. P1：个人中心与学习数据

> **优先级**：🟡 中 — P0 完成后立即进行
> **参考原型**：[02-学习复盘与个人中心.html](./02-学习复盘与个人中心.html) Page 1、Page 2

### 4.1 个人中心主页

对应原型图 **Page 1: 个人中心**。

**展示内容**：

```
┌──────────────────────────────┐
│  [头像]  昵称                 │
│           Lv.X · 等级称号     │
│           ████████░░ 经验条   │
├──────────────────────────────┤
│   47        82%       12     │
│  累计闯关   正确率    连续天数  │
├──────────────────────────────┤
│  学习工具                     │
│  ┌────┬────┬────┐            │
│  │学习│错题│成就│            │
│  │历史│本  │墙  │            │
│  ├────┼────┼────┤            │
│  │打卡│排行│收藏│            │
│  │日历│榜  │夹  │            │
│  └────┴────┴────┘            │
├──────────────────────────────┤
│  灯灯说：今天也要加油闯关哦！   │
└──────────────────────────────┘
```

**用户统计指标**（实时计算）：
| 指标 | 计算方式 |
|------|---------|
| 累计闯关 | `COUNT(quiz_sessions WHERE user_id = ?)` |
| 正确率 | `AVG(accuracy)` 所有闯关的平均正确率 |
| 连续天数 | 从今天往前数，连续有打卡/闯关记录的天数 |
| 金币余额 | `users.coins` 字段 |
| 等级 | 根据累计经验值（答对题数×10 + 闯关次数×5）计算 |

**等级体系**：

| 等级 | 称号 | 所需经验 |
|------|------|---------|
| Lv.1 | 初学萌新 | 0 |
| Lv.2 | 知识学徒 | 100 |
| Lv.3 | 学习达人 | 300 |
| Lv.4 | 百科高手 | 600 |
| Lv.5 | 博学大师 | 1000 |
| ... | ... | 每级递增 `level × 200` |

### 4.2 学习历史

对应原型图 **Page 2: 学习历史**。

**列表项字段**：
| 字段 | 说明 |
|------|------|
| 知识领域图标 | 缩写字母（Py/史/Js...） |
| 闯关标题 | 如"Python面试高频题50道" |
| 日期 | 闯关完成时间 |
| 题目数 | 本次闯关总题数 |
| 用时 | 总耗时 |
| 得分 | 如 9/10，按分数颜色标记（绿→好，黄→中，红→差） |

**筛选功能**：
- 全部 / 本周 / 本月
- 按知识领域筛选（动态从历史数据中提取）

**分页**：
- 每页 20 条，下拉加载更多

### 4.3 闯关记录自动保存

**触发时机**：每次调用 `POST /api/quiz/analyze` 后，若请求携带有效 JWT Token，则自动：

1. 在 `quiz_sessions` 表插入一条闯关记录
2. 将错题写入 `wrong_questions` 表
3. 累计用户金币（见积分规则）
4. 累计用户经验值
5. 写入当日打卡记录（`check_ins` 表）

**未登录用户**：
- 闯关功能正常使用，但数据不保存
- 结果页底部提示"登录后可保存学习记录"

## 5. P2：错题本与学习激励

> **优先级**：🟢 中低 — P1 完成后进行
> **参考原型**：[02-学习复盘与个人中心.html](./02-学习复盘与个人中心.html) Page 3、Page 4

### 5.1 错题本列表

对应原型图 **Page 3: 我的错题本**。

**展示方式**：
- 按知识领域分组（如 Python、中国近代史、JavaScript）
- 每组显示错题数量
- 顶部统计"共有 X 道错题待复习"

**单条错题信息**：
| 字段 | 说明 |
|------|------|
| 题目内容 | 截断显示，不超过 2 行 |
| 你的答案 | 红色标记 |
| 正确答案 | 绿色标记 |
| 来源闯关 | 属于哪次闯关 |

**功能**：
- 点击进入详情
- "全部重做"按钮 → 将所有错题组成一次新闯关

### 5.2 错题详情与重做

对应原型图 **Page 4: 错题详情**。

**展示内容**：
- 题目完整内容 + 类型 + 难度
- 上次错误答案 vs 正确答案对比
- 灯灯讲解（详细解析）
- 四个选项（可重新选择）
- 选对后"掌握！从错题本移除"按钮

**交互流程**：
```
查看错题 → 重新选择答案
  → 选对：弹窗"已掌握"，标记 resolved=true
  → 选错：弹窗"继续加油"，保持待复习状态
```

### 5.3 每日打卡

对应原型图 **Page 6: 学习打卡日历**（简化版）。

**打卡规则**：
- 用户当天完成至少 1 次闯关 → 自动打卡
- 连续打卡天数展示在个人中心
- 断签后重新计数

**数据存储**：
- `check_ins` 表：`(user_id, check_date)` 唯一约束

## 6. 积分/金币规则

### 6.1 获取金币

| 行为 | 金币 |
|------|------|
| 完成一次闯关（基础） | +10 |
| 每答对 1 题（额外） | +2 |
| 连续答对 ≥5 题（Combo Bonus） | +5 |
| 连续答对 ≥10 题（Super Combo） | +15 |
| 每日首次闯关（首胜） | +20 |

**单次闯关金币上限**：100 金币

**示例计算**：
- 10 题闯关，答对 8 题，最高连击 7：
  `10(基础) + 8×2(正确) + 5(combo≥5) = 31 金币`

### 6.2 金币用途（后续迭代预留）

> 本期不实现金币消费功能，仅做累积展示。

后续可能的消费场景：
- 兑换"双倍经验卡""提示卡""免答卡"等道具
- 解锁特殊 AI 伙伴皮肤
- 解锁高级知识领域

## 8. 验收标准

### P0 验收

- [ ] 微信小程序端可完成 `wx.login` → 后端获取 openid → 返回 JWT
- [ ] H5 端可输入昵称完成 Mock 登录
